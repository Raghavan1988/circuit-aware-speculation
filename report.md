# Acceptance-Aware Speculation: Results Summary

Prepared 2026-07-12. Models: Qwen2.5-7B-Instruct (target) with Qwen2.5-0.5B-Instruct (draft). Compute: Modal, A100.

## Summary

We set out to study speculative decoding and answer two questions: can we predict which draft tokens a large model will accept, and is it worth adapting the draft length per step. We built a verified decoding engine, generated a full labelled corpus, and ran the analysis end to end. The headline results are: (1) acceptance has clean, reproducible structure by token type and position, which is a solid and useful finding; (2) cheap, free-to-compute signals predict acceptance well (AUROC around 0.84 to 0.87), and a linear probe on the draft model's internal states does not beat them, which is a clear negative result; and (3) the speed benefit of adaptive routing is marginal on our research harness because of a hardware effect (the small draft model is not actually cheap to run one token at a time), and this is fixable but is deferred engineering. The net effect is a coherent, honest contribution centered on the acceptance atlas plus a rigorous demonstration that cheap signals are a strong baseline.

## 1. Background

Speculative decoding speeds up a large "target" model by having a small "draft" model propose several tokens that the target then verifies in parallel. It is lossless: the output is identical to running the target alone. It only pays off if the draft is cheap and its tokens are usually accepted. Two questions follow naturally:

- Prediction question (science): can we tell in advance whether a draft token will be accepted, and does the draft model's internal state carry information beyond simple signals like its own confidence. This is measured in prediction accuracy (AUROC) and does not involve timing.
- Routing question (systems): does choosing the draft length per step beat using one fixed length. Because the output is fixed no matter what, the only thing routing can change is speed, so this question is inseparable from latency.

## 2. What we built and verified

- An exact greedy speculative-decoding engine, proven token-identical to running the target alone. In full precision it passed the equivalence gate 73 out of 73, including a critical fix (found by an adversarial internal review) to how the engine stops at the end-of-sequence token.
- A full corpus: 644 prompts drawn from four standard public datasets (code, math, chat, summarization), decoded under 8 policies (no speculation, plus fixed draft lengths 1 through 8), sealed as immutable data with checksums. This produced about 302,000 labelled draft tokens.
- A measurement-first discipline: every reported number is generated by a script from the sealed data. Nothing is hand-entered.

## 3. Main results

### 3.1 The acceptance atlas (positive finding)

We measured acceptance rate for every combination of token category (for example code punctuation, numbers, named entities, function words) and generation phase (early, middle, late), with confidence intervals from resampling prompts. The result is clean and legible across 38 cells:

- Code-like tokens are accepted most often: code delimiters, operators, and numbers land around 0.85 to 0.88.
- Content-bearing tokens are hardest: named entities and reasoning-transition words sit around 0.58 to 0.67.

In plain terms, whether a draft token is likely to be accepted depends strongly on what kind of token is coming next. This is a reusable, model-agnostic-looking finding and is the strongest part of the work. It is independent of any timing question.

### 3.2 Predicting acceptance: cheap signals are strong

We fit simple, regularized predictors of acceptance using prompt-grouped cross-validation (so no prompt leaks between training and testing). Using only free signals that the draft already produces (its own entropy, its top-1 margin, and recent acceptance history), the predictor reaches AUROC about 0.84 on the full development set. This is a strong baseline, and it costs nothing extra to compute because those signals are byproducts of drafting.

### 3.3 Do the model's internal states help? The decisive test (negative finding)

The central scientific question was whether the draft model's hidden internal states carry acceptance information beyond those cheap signals. To test this we captured the draft's internal representations at four depths (early, middle, late, final) for 42,568 tokens, with a capture method that guarantees exact alignment between each activation and its token (validated at 99.2 percent). We then fit linear probes and compared.

| Predictor | AUROC |
|---|---|
| Cheap surface signals (the bar to beat) | 0.870 |
| Hidden states only, best layer (layer 18) | 0.803 |
| Hidden states plus surface signals, best layer | 0.876 |

The finding is clear: a linear probe on hidden states does not beat the cheap signals (0.803 is well below 0.870), and adding hidden states on top of the cheap signals improves things by at most 0.006 AUROC, which is negligible. For this model pair, the free signals already capture the linearly-accessible acceptance information.

This is a genuine and useful negative result. It answers the pivotal question we set out to test, it is publishable in its own right (well-run null results are valuable), and it puts recent related work in context: a recently published linear "acceptance head" on draft states is, here, matched by signals that cost nothing. We also see a sensible secondary pattern: acceptance information becomes slightly more linearly accessible in deeper layers (rising to a peak at layer 18), just never enough to overtake the cheap baseline.

### 3.4 The speed question and a hardware finding

When we measured the actual cost of routing, we found that adaptive draft-length selection buys only about 5 percent on our harness, and the best fixed policy is in fact "do not speculate at all." Investigating why, we found the cause is not the algorithm but a hardware effect: running the small 0.5B draft one token at a time costs about 24 ms per token, almost the same as one pass of the 7B target (about 30 ms). The reason is that single-token decoding is "launch-bound," meaning its cost is dominated by the fixed overhead of dispatching many small GPU operations, which scales with the number of model layers rather than the model size. So the small draft is not actually cheap in this setting, and speculation barely breaks even.

Two points matter here. First, this is a property of the research harness (chosen for flexibility and for the ability to read internal states), not a fundamental limit. With the same acceptance data, if the draft ran at a realistic 2 to 8 ms per token, the routing benefit would be 25 to 46 percent. Second, making the draft that fast requires production-style engineering (a fixed-size cache plus GPU graph capture) that we have deliberately deferred. We built and tested the switch that would accept such a fast path, and we confirmed that simpler shortcuts (fused attention, standard compilation) do not solve it. All of this is recorded.

## 4. What this means for the contribution

The work now tells a clear and honest story:

1. Acceptance has strong structure by token category and phase (the atlas). Positive, novel, reusable.
2. Cheap surface signals are a strong, hard-to-beat baseline for acceptance prediction, and linear probes on hidden states do not beat them. A rigorous negative result that saves others effort and contextualizes recent related work.
3. On a flexibility-first research harness the small draft is launch-bound, so the routing speedup is marginal; the real opportunity is real but sits behind deferred serving-grade engineering.

Importantly, points 1 and 2 do not depend on the timing issue at all, so they stand regardless of the hardware story. The reusable corpus itself is a citable artifact.

## 5. Risks and open questions

- The negative probe result is currently for a linear probe, four layers, one model pair, on the development split. A nonlinear probe, other layers, or a different model family could in principle change it. We scope the claim accordingly.
- The prediction result should be checked on a second model family to show it generalizes (see replication status below).
- The speed opportunity (25 to 46 percent) is a model estimate from the acceptance data, not yet a measured wall-clock result; measuring it requires the deferred fast-draft path.
- One data-quality note: an early split-labeling bug left a metadata column unusable, but it is fully recoverable at analysis time and does not affect any result. It will be corrected before the corpus is published.

## 6. Next steps

- Near term, low risk: package the corpus as a citable dataset and write up the atlas and the negative probe result. This is the safe, done core and needs no further GPU work.
- Second model family: replicate the atlas and the probe comparison on a Llama pair to show the findings generalize (blocked on access, see below).
- Deferred, higher ceiling: build the fast-draft path (fixed-size cache plus GPU graph capture) to realize the routing speedup. This is real but bounded engineering, reserved for later.

## 7. Replication status (second model family)

We wired the harness to switch model families with a single setting and prepared to replicate on the Llama 3.1 8B and Llama 3.2 1B pair. Access to the 8B target appears to work, but the 1B draft model is a separate gated repository and its access is not yet granted, so the run is currently blocked at that step. Once access to the 1B draft is approved, replication of the gate, the atlas, and the probe comparison can proceed immediately.
